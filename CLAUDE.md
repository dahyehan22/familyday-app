# FamilyDay

## 프로젝트 개요

가족 스케줄 & 할일 관리 모바일 앱 (React 18 + Vite)
모바일 퍼스트 (390×844), 한국어 전용 UI

## 명령어

- `pnpm dev` — 개발 서버 (localhost:3000)
- `pnpm build` — 프로덕션 빌드
- `pnpm preview` — 빌드 미리보기

## 코드 컨벤션

### 구조

- 모든 컴포넌트는 `src/FamilyDay.jsx` 단일 파일에 작성
- 새 컴포넌트 추가 시 기존 패턴(함수 컴포넌트, props 구조)을 따를 것
- React Fragment (`<>...</>`) 대신 `<div>` 사용

### 스타일링

- CSS-in-JS 인라인 스타일만 사용 (별도 CSS 파일 금지)
- 색상은 반드시 `C` 토큰 객체 사용 (하드코딩 HEX 금지)
- 새 키프레임 애니메이션은 `<style id="fd-kf">` 블록에 추가
- borderRadius: 카드 20px, 버튼/뱃지 999px, 모달 28px 28px 0 0
- boxShadow: 카드 `0 4px 20px rgba(108,99,255,.08)`

### 텍스트

- 모든 UI 텍스트는 한국어로 작성
- 폰트: Nunito (Google Fonts, 동적 로드)
- 타이포: 헤딩 18-22px/900, 본문 14-15px/600, 캡션 11-13px/700

### 데이터

- 상태 관리: React useState만 사용 (외부 라이브러리 없음)
- ID 생성: `uid()` 함수 사용
- 날짜 포맷: `YYYY-MM-DD` 문자열

## 문서 관리

### FamilyDay-기능스펙

- 새로운 기능이 추가되거나 기존 기능의 스펙이 변경되면 반드시 `FamilyDay-기능스펙` 문서를 업데이트할 것
- 기능 추가/수정/삭제 시 해당 문서에 변경 사항을 반영

### FamilyDay-디자인시스템

- 디자인이 변경되거나 새로운 디자인 요소가 추가되면 반드시 `FamilyDay-디자인시스템` 문서를 업데이트할 것
- 색상 토큰, 타이포그래피, 컴포넌트 스타일, 애니메이션 등 디자인 관련 변경 시 해당 문서에 반영

### FamilyDay-개발계획

- 앞으로 추가할 기능이나 개선 사항이 논의되면 반드시 `FamilyDay-개발계획.md` 문서에 업데이트할 것
- 새로운 기능 아이디어, 예정된 개발 항목, 우선순위 변경 시 해당 문서에 반영
- 기능 개발이 완료되면 해당 항목의 상태를 업데이트할 것
